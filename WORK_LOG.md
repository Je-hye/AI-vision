# 작업 로그 (WORK_LOG.md)

## 프로젝트 개요
실시간 비디오 스트림에서 손동작(Gesture) 이벤트를 감지하고 대시보드에 표시하는 시스템입니다.

## 완료된 작업
1.  **프로젝트 구조 설정**: FastAPI 기반의 백엔드 및 정적 파일 구조 설계.
2.  **백엔드 API 구현 (`app/main.py`)**:
    *   상태 확인, 스트림 시작/중지, 이벤트 조회 엔드포인트 구현.
3.  **Roboflow 연동 (`app/roboflow_client.py`)**:
    *   Roboflow Hosted API를 사용하여 객체 탐지 기능 통합.
4.  **스트림 처리기 (`app/stream_processor.py`)**:
    *   OpenCV를 사용한 RTSP/HLS/HTTP 스트림 프레임 캡처 및 처리 로직.
5.  **이벤트 처리 및 저장 (`app/event_rules.py`, `app/event_store.py`)**:
    *   연속 프레임 감지 조건, 쿨다운 시간 등을 적용한 이벤트 규칙 정의.
    *   JSONL 형식의 이벤트 로그 및 스냅샷 저장 기능.
6.  **프론트엔드 대시보드 (`app/static/`)**:
    *   실시간 프레임 출력, 감지 결과 및 이벤트 로그 표시를 위한 HTML/JS/CSS.
7.  **유틸리티 및 테스트**:
    *   `realtime/rtsp_optical_flow.py`: 광학 흐름(Optical Flow) 분석 도구.
    *   `tests/`: 이벤트 규칙 및 Roboflow 클라이언트 단위 테스트.

## 현재 상태
*   기본적인 손동작 감지 및 대시보드 기능이 동작하는 초기 프로토타입 상태입니다.

## 최근 이슈 해결 (2026-05-20)
*   **서버 시작 오류 수정**: `app/stream_processor.py`의 깨진 문자열 및 중복 `_set_error` 정의를 제거하고, 누락된 `cv2` import를 추가하여 FastAPI 앱이 정상 import되도록 수정.
*   **프레임 표시 개선**: Roboflow API 인증/요청 오류가 발생해도 최근 캡처 프레임은 대시보드에 표시되도록 스트림 처리 순서를 조정.
*   **렌더링 지연 개선**: 영상 프레임 갱신과 Roboflow 분석 요청을 분리하고, 대시보드 상태 갱신 주기를 단축하여 API 응답 지연이 화면 렌더링을 막지 않도록 개선.

## 최근 개선 작업 (2026-05-21)
*   **API 키 노출 방지**: Roboflow 오류 메시지에서 `api_key` 값을 마스킹하고 401/403/429/5xx 상태별 안내 메시지를 추가.
*   **프레임 처리 최적화**: 최신 미리보기 프레임은 메모리 JPEG로 서빙하고, 분석용 임시 파일은 이벤트 스냅샷이 아니면 삭제하도록 개선.

*   **작업 문서 정리**: 남은 보완 작업을 `TODO.md`에 정리하고, 실행 중 확인된 에러와 해결 방법을 `TROUBLESHOOTING.md`에 추가.
